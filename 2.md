# PARTE 1: MÉTRICAS DE TRAINING

## 1. Loss (Pérdida)

### ¿Qué es?
Es la **función de error** que el modelo intenta minimizar durante el entrenamiento.

### Fórmula General
```
Loss = suma de penalizaciones por predicción incorrecta
Cada predicción contribuye una penalización basada en qué tan lejos está de lo correcto
```

### Tipos de Loss Usados en Nuestros Experimentos

#### A) Binary Crossentropy (Experimentos 1 y 2 - Multi-etiqueta)

**Usado para**: Cada uno de los 5 dedos se trata independientemente
- ¿Está el pulgar extendido? SÍ/NO
- ¿Está el índice extendido? SÍ/NO
- etc.

**Fórmula**:
```
BCE = -[y*log(ŷ) + (1-y)*log(1-ŷ)]

donde:
  y = etiqueta verdadera (0 o 1)
  ŷ = probabilidad predicha (0 a 1)
```

**Ejemplo interpretativo**:
```
Caso 1: El dedo ESTÁ extendido (y=1)
├─ Modelo predice 0.95 → BCE ≈ 0.051 (penalización baja - bien)
├─ Modelo predice 0.50 → BCE ≈ 0.693 (penalización media - mal)
└─ Modelo predice 0.10 → BCE ≈ 2.303 (penalización alta - muy mal)

Caso 2: El dedo NO está extendido (y=0)
├─ Modelo predice 0.05 → BCE ≈ 0.051 (bien)
├─ Modelo predice 0.50 → BCE ≈ 0.693 (mal)
└─ Modelo predice 0.90 → BCE ≈ 2.303 (muy mal)
```

**Interpretación en contexto**:
- Exp 1: Loss final 0.0110 → Promedio de error por dedo muy bajo
- Exp 2: Loss final 0.6526 → Promedio de error sigue siendo alto (mal)

#### B) Categorical Crossentropy (Experimentos 3 y 4 - Multiclase)

**Usado para**: Una sola clase entre 27 opciones mutuamente excluyentes
- La imagen es EXACTAMENTE UNA de: 0, 1, 2, ..., yes

**Fórmula**:
```
CCE = -Σ(y_i * log(ŷ_i))

donde:
  y_i = 1 si la clase i es correcta, 0 en otro caso
  ŷ_i = probabilidad predicha para clase i
```

**Ejemplo interpretativo**:
```
Imagen: Dígito "5" (clase verdadera = 5)

Predicción 1 (BUENA):
  Clase 5: 0.92 → log(0.92) ≈ -0.083 
  Loss ≈ 0.083 (bajo - bien)

Predicción 2 (MALA):
  Clase 5: 0.30 → log(0.30) ≈ -1.204
  Loss ≈ 1.204 (alto - mal)

Predicción 3 (TERRIBLE):
  Clase 5: 0.01 → log(0.01) ≈ -4.605
  Loss ≈ 4.605 (muy alto - muy mal)
```

**Interpretación en contexto**:
- Exp 3: Loss final 0.0450 → Promedio de confianza muy bajo
- Exp 4: Loss final 0.1642 → Promedio de confianza mayor (peor que Exp 3)

### 📌 Regla General de Interpretación de Loss

```
Loss < 0.10    → Excelente (modelo muy confiado y correcto)
Loss 0.10-0.30 → Bueno
Loss 0.30-0.50 → Aceptable
Loss 0.50-1.00 → Débil
Loss > 1.00    → Deficiente
```

**Importante**: El valor absoluto de loss **solo es comparable dentro del mismo experimento**. No compares Loss entre Exp 1 y Exp 3 directamente.

## 2. Accuracy (Exactitud)

### ¿Qué es?
Proporción de predicciones **completamente correctas**.

### Fórmula
```
Accuracy = (Predicciones Correctas) / (Total de Predicciones)
         = 0 a 1 (o 0% a 100%)
```

### Tipos de Accuracy

#### A) Binary Accuracy (Multi-etiqueta)
Mide si cada dedo se clasificó correctamente independientemente.

**Ejemplo**:
```
5 dedos, imagen 1:
  Verdadero:   [1, 1, 0, 1, 0]  (pulgar, índice, anular extendidos)
  Predicción:  [1, 1, 0, 1, 0]  → Binary Accuracy = 5/5 = 1.0 ✅
  
5 dedos, imagen 2:
  Verdadero:   [1, 1, 0, 1, 0]
  Predicción:  [1, 1, 0, 0, 0]  → Binary Accuracy = 4/5 = 0.8 ⚠️ (falló anular)
```

**Contexto**: 
- Exp 1: Binary Accuracy (train) = 0.6101 → 61% de dedos correctos (solo al inicio)
- Exp 1: Binary Accuracy (final) = 0.9966 → 99.66% de dedos correctos

#### B) Categorical Accuracy (Multiclase)
Mide si la clase predicha es exactamente la verdadera.

**Ejemplo**:
```
27 clases posibles (0-9, a-z, palabras)
  Verdadero:   "5" (dígito cinco)
  Predicción:  "5" → Accuracy = 1.0 ✅
  
  Verdadero:   "5"
  Predicción:  "3" → Accuracy = 0.0 ❌ (totalmente incorrecto, aunque cercano)
```

**Contexto**:
- Exp 3: Accuracy (train) = 0.0854 → Solo 8.54% correctas al inicio (esperado)
- Exp 3: Accuracy (final test) = 0.9845 → 98.45% de clases predichas correctamente

#### C) Top-3 Accuracy (Multiclase)
¿La clase verdadera está entre las **top-3 probabilidades predichas**?

**Ejemplo**:
```
Verdadero: "5"
Probabilidades predichas: 0:"0.05", 1:"0.03", 3:"0.15", 5:"0.45", 7:"0.32"
Top-3: [5 (0.45), 7 (0.32), 3 (0.15)]
→ Top-3 Accuracy = 1.0 ✅ (5 está en top-3)

Verdadero: "5"
Top-3: [7 (0.35), 3 (0.30), 0 (0.25)]
→ Top-3 Accuracy = 0.0 ❌ (5 no está en top-3)
```

**Interpretación**:
- Exp 3: Top-3 = 0.9991 → 99.91% de veces, la clase verdadera está en top-3
- Indica que el modelo **casi siempre** tiene la clase correcta en sus "top 3 candidatos"

## 3. Training vs Validation Loss/Accuracy

### Conceptos Clave

```
Training Set  → Datos usados para entrenar (el modelo "ve" estos)
Validation Set → Datos diferentes, usados para monitorear (el modelo "no ve" estos)
```

### ¿Qué Indica la Relación?

#### Caso A: Train ≈ Val (Equilibrio - Ideal)
```
Ejemplo (Exp 1):
  Train Loss: 0.0110
  Val Loss:   0.0454
  Brecha:     0.0343 ← Pequeña
  
  Interpretación:
  ✅ Modelo aprendió bien Y generaliza bien
  ✅ No hay overfitting (memorizó train)
  ✅ No hay underfitting (no aprendió suficiente)
```

#### Caso B: Train << Val (Overfitting - Malo)
```
Ejemplo hipotético:
  Train Loss: 0.001
  Val Loss:   1.500
  Brecha:     1.499 ← MUY Grande
  
  Interpretación:
  ❌ Modelo memorizó datos de entrenamiento
  ❌ No generaliza a datos nuevos
  ❌ Problema: demasiadas épocas, poca regularización
```

#### Caso C: Train ≈ Val pero ambos altos (Underfitting - Malo)
```
Ejemplo (similar a Exp 2):
  Train Loss: 0.6097
  Val Loss:   0.6752
  Brecha:     0.0648 ← Pequeña pero ambas ALTAS
  
  Interpretación:
  ❌ Modelo NO aprendió suficientemente
  ❌ Arquitectura muy pequeña O dataset insuficiente
  ❌ Problema: pocas épocas, baja capacidad
```

# PARTE 2: MÉTRICAS DE EVALUACIÓN EN TEST

## 1. Exact Match Ratio (EMR) - Multi-etiqueta

### ¿Qué es?
Porcentaje de **todas las 5 etiquetas correctas simultáneamente**.

### Fórmula
```
EMR = (Muestras donde los 5 dedos son correctos) / (Total muestras)
```

### Ejemplo
```
Muestra 1:
  Verdadero:   [Pulgar✓, Índice✓, Medio✓, Anular✓, Meñique✓]
  Predicción:  [Pulgar✓, Índice✓, Medio✓, Anular✓, Meñique✓]
  → Contribuye a EMR = SÍ ✅

Muestra 2:
  Verdadero:   [Pulgar✓, Índice✓, Medio✗, Anular✓, Meñique✓]
  Predicción:  [Pulgar✓, Índice✓, Medio✓, Anular✓, Meñique✓]
  → Contribuye a EMR = NO ❌ (aunque 4/5 correctas)
  
1298 muestras totales
1272 con todos los 5 correctos
EMR = 1272/1298 = 0.9815 (98.15%)
```

### Interpretación
- Exp 1: **0.9815** = De 1,298 test samples, 1,272 tienen los 5 dedos perfectos
- Métrica **muy estricta**: un error en cualquier dedo cuenta como fracaso total
- Útil para aplicaciones críticas (ej: lenguaje de signos para sordomudos)

### Umbrales en Nuestros Experimentos
```
Umbral aceptable: 0.85 (85%)
Umbral robusto:   0.92 (92%)

Exp 1: 0.9815 >> 0.92 ✅ ROBUSTO
Exp 2: 0.1002 << 0.85 ❌ FRACASO TOTAL
```

## 2. Hamming Accuracy

### ¿Qué es?
Porcentaje **promedio de predicciones correctas por dedo** (menos estricto que EMR).

### Fórmula
```
Hamming Accuracy = (Dedos predichos correctamente) / (Total de dedos)
                 = promedio de accuracy sobre cada dedo
```

### Ejemplo
```
Muestra 1: [✓, ✓, ✓, ✓, ✓] → Contribuye 5/5
Muestra 2: [✓, ✓, ✗, ✓, ✓] → Contribuye 4/5
Muestra 3: [✗, ✗, ✓, ✓, ✓] → Contribuye 3/5

Hamming = (5 + 4 + 3) / (5*3) = 12/15 = 0.80
```

### Interpretación
- Exp 1: **0.9935** = En promedio, 99.35% de los dedos se predice correctamente
- Más **tolerante** que EMR: penaliza solo dedos incorrectos individuales
- Útil para identificar si hay un dedo particularmente difícil

### Comparación con EMR (Exp 1)
```
EMR:               0.9815 (98.15%)  ← Todos 5 correctos simultáneamente
Hamming Accuracy:  0.9935 (99.35%)  ← En promedio por dedo
Diferencia:        0.012 (1.2%)
Interpretación:    Del 1.8% de muestras fallidas en EMR,
                   la mayoría tienen solo 1-2 dedos incorrectos
```

## 3. Precision (Precisión)

### ¿Qué es?
De **todas las predicciones POSITIVAS** que hizo el modelo, ¿cuántas fueron correctas?

### Fórmula
```
Precision = True Positives / (True Positives + False Positives)
          = Predicciones correctas / Todas mis predicciones positivas
```

### Ejemplo (Multi-etiqueta - Pulgar)

```
Verdadero estado del pulgar en 100 imágenes:
  Extendido en 60 imágenes
  Contraído en 40 imágenes

Mi predicción: "Pulgar extendido" en 70 imágenes

Matriz de confusión:
                  Verdadero Extendido  Verdadero Contraído
  Predicho Extendido:    58 (TP)            12 (FP)
  Predicho Contraído:     2 (FN)            28 (TN)

Precision = TP / (TP + FP) = 58 / 70 = 0.829 (82.9%)
```

**Interpretación**: "De los 70 casos donde predije pulgar extendido, 58 estaban correctos"

### Ejemplo (Multiclase - Clase "5")

```
Verdaderos: 130 imágenes de clase "5"

Mi predicción: "5" en 132 imágenes
  Correctas: 128
  Incorrectas: 4 (fueron otras clases)

Precision = 128 / 132 = 0.970 (97.0%)
```

**Interpretación**: "De los 132 casos donde predije clase 5, 128 fueron correctos"

### ¿Cuándo importa?

```
Casos de alto costo de FALSA ALARMA:
├─ Diagnóstico médico: Precisión crítica (falsos positivos = alarmas falsas)
├─ Spam detection: Precisión importante (no queremos marcar emails legítimos como spam)
└─ ASL reconocimiento: Menos crítica (traducción errada < sin respuesta)
```

### Contexto en Nuestros Experimentos

```
Exp 1, Pulgar: Precision = 0.994
→ "De 994 detecciones de pulgar, 988 fueron correctas" (4 falsos positivos)

Exp 3, Clase "0": Precision = 0.99
→ "De 99 predicciones de '0', todas fueron correctas" (0 falsos positivos)
```

## 4. Recall (Sensibilidad/Recuperación)

### ¿Qué es?
De **todas las instancias POSITIVAS reales**, ¿cuántas fueron encontradas?

### Fórmula
```
Recall = True Positives / (True Positives + False Negatives)
       = Casos encontrados / Todos los casos reales
```

### Ejemplo (Multi-etiqueta - Pulgar)

```
Verdadero: Pulgar extendido en 60 imágenes
Mi predicción: "Pulgar extendido" en 70 imágenes
  Correctas: 58
  Fallos: 2 (no detecté 2 pulgares extendidos)

Recall = 58 / 60 = 0.967 (96.7%)
```

**Interpretación**: "De los 60 pulgares realmente extendidos, detecté 58"

### Ejemplo (Multiclase)

```
Verdaderos: 130 imágenes de clase "5"
Mi predicción: "5" en 132 imágenes
  Correctas: 128
  Fallos: 2 (no detecté 2 imágenes de clase 5)

Recall = 128 / 130 = 0.985 (98.5%)
```

**Interpretación**: "De las 130 imágenes que eran clase 5, encontré 128"

### ¿Cuándo importa?

```
Casos de alto costo de FALSOS NEGATIVOS:
├─ Detección de cáncer: Recall crítica (perder un caso = muerte)
├─ Detección fraude: Recall importante (perder fraude = pérdida financiera)
├─ Sistemas de seguridad: Recall importante (no detectar amenaza = peligro)
└─ ASL reconocimiento: Moderadamente importante (no traducir = pérdida de mensaje)
```

---

## 5. F1-Score

### ¿Qué es?
**Media armónica** entre Precision y Recall. Un balance entre ambas.

### Fórmula
```
F1 = 2 * (Precision * Recall) / (Precision + Recall)
   = Rango 0 a 1 (higher is better)
```

### Ejemplo

```
Pulgar (Exp 1):
  Precision: 0.994
  Recall:    0.999
  F1 = 2 * (0.994 * 0.999) / (0.994 + 0.999)
     = 2 * 0.993 / 1.993
     = 0.996

Interpretación: Balance perfecto entre precision y recall
```

### ¿Cuándo usar F1?

```
✅ Cuando ambas métricas importan igual
  - Clasificación desbalanceada (diferentes cantidad de muestras por clase)
  - Necesitas métrica única resumen

❌ Cuando una métrica importa más que otra
  - Si Recall >> Precision: usa Recall
  - Si Precision >> Recall: usa Precision
```

### Contexto

```
Exp 1:
  Macro F1-Score = 0.9927
  → Promedio de F1 sobre los 5 dedos = 99.27%

Exp 3:
  Macro F1-Score = 0.9822
  → Promedio de F1 sobre las 27 clases = 98.22%
```

## 6. Macro vs Weighted Average

### Tipos de Promediado

```
MACRO = Promedio simple (todas las clases importan igual)
WEIGHTED = Promedio ponderado (clases con más muestras pesan más)
```

### Ejemplo (Exp 3: Multiclase con clases desbalanceadas)

```
Clase NULL:        47 muestras, F1=0.89
Clase "0":        130 muestras, F1=0.99
Clase "goodbye":  130 muestras, F1=1.00
... (25 clases más)

MACRO F1 = (0.89 + 0.99 + 1.00 + ... + otros) / 27
         = Trata todas igual, aunque NULL tenga pocas muestras

WEIGHTED F1 = (0.89*47 + 0.99*130 + 1.00*130 + ...) / 3421
            = Da más peso a clases frecuentes
```

### Interpretación

```
Si MACRO >> WEIGHTED:
  → Hay clases con pocas muestras que funcionan bien
  
Si MACRO ≈ WEIGHTED:
  → Dataset equilibrado, ambas métricas similares
  
Si MACRO << WEIGHTED:
  → Clases raras funcionan mal, pero el modelo brilla en clases frecuentes
```

### En Nuestros Experimentos

```
Exp 3:
  Macro F1:    0.9822 (98.22%)
  Weighted F1: 0.9800 (98.00%)
  Diferencia:  0.22%
  
  Conclusión: Dataset bastante equilibrado (pocos decimales de diferencia)
```

# PARTE 3: MATRIZ DE CONFUSIÓN (Conceptual)

## ¿Qué es?

Tabla que muestra todos los tipos de predicciones:

```
                    VERDADERO POSITIVO  VERDADERO NEGATIVO
                         (TP)                  (TN)
PREDICHO POSITIVO     Acertó (Sí)        Falsa Alarma (FP)
PREDICHO NEGATIVO  Perdió caso (FN)     Acertó (No)
```

## Ejemplo (Pulgar - Multi-etiqueta)

```
                        Pulgar Real = SÍ  Pulgar Real = NO
Pulgar Predicho = SÍ:        988 (TP)          12 (FP)
Pulgar Predicho = NO:          2 (FN)         296 (TN)

De estas 4 celdas:
├─ TP (988): "Predije SÍ y era SÍ" ✅
├─ TN (296): "Predije NO y era NO" ✅
├─ FP (12): "Predije SÍ pero era NO" ❌ (falsa alarma)
└─ FN (2): "Predije NO pero era SÍ" ❌ (se perdió un caso)

Precision = TP / (TP+FP) = 988/1000 = 0.988
Recall = TP / (TP+FN) = 988/990 = 0.998
```

# PARTE 4: MÉTRICAS DE CONVERGENCIA

## 1. Epoch

### ¿Qué es?
Un ciclo completo de entrenamiento por todo el dataset.

### Ejemplo
```
Dataset: 1,000 imágenes
Batch size: 32

1 Epoch = 1000/32 = 31.25 → 32 pasos (batches)

En Exp 1:
├─ Epoch 1: procesa muestras 0-6058, calcula Loss
├─ Epoch 2: procesa mismas 6058 muestras NUEVAMENTE, calcula Loss
├─ ...
└─ Epoch 60: último ciclo, modelo finalmente "ve" todas 60 veces
```

### Número de Épocas Usado

```
Exp 1: 60 épocas (pero paró en epoch 50 por EarlyStopping)
Exp 3: 60 épocas (pero paró en epoch 59 por EarlyStopping)
Exp 4: 35 épocas Fase 2 (paró en epoch 34 por EarlyStopping)
```

## 2. Learning Rate (Tasa de Aprendizaje)

### ¿Qué es?
Qué tan grande son los "pasos" que da el modelo para minimizar loss.

### Fórmula Simple
```
new_weights = old_weights - learning_rate * gradient
```

### Ejemplo Visual

```
LR = 0.1 (Grande - SALTOS GRANDES)
  Iteración 1: Loss = 2.0 → Salta a 1.5
  Iteración 2: Loss = 1.5 → Salta a 1.2
  ❌ Puede saltar el mínimo

LR = 0.001 (Pequeño - PASOS PEQUEÑOS)
  Iteración 1: Loss = 2.0 → Salta a 1.99
  Iteración 2: Loss = 1.99 → Salta a 1.98
  ✅ Más preciso pero lento
```

### En Nuestros Experimentos

```
Exp 1-3: LR = 0.001 (1e-3)     → Moderado
Exp 2, Fase 1: LR = 0.001      → Igual
Exp 2, Fase 2: LR = 1e-5       → 100x MENOR (muy conservador)
Exp 4, Fase 2: LR = 1e-5       → 100x MENOR

Motivo del LR bajo en Phase 2:
  "No queremos destruir el aprendizaje de ImageNet"
```

### ReduceLROnPlateau

```
Mecanismo: Si loss no mejora en N épocas, dividir LR por 10

Ejemplo (Exp 1):
  Epoch 37: Loss no mejora → LR: 0.001 → 0.0005
  Epoch 55: Loss no mejora → LR: 0.0005 → 0.00025
  
Objetivo: Hacer "pasos más pequeños" cuando está atascado
```

# PARTE 5: REGULARIZACIÓN Y VALIDACIÓN

## 1. Early Stopping

### ¿Qué es?
Detener el entrenamiento cuando la validación deja de mejorar.

### Ejemplo

```

────────────────────────────────────
1      0.70        0.75      Mejor val_loss, guarda modelo ✅
2      0.65        0.72      Mejor val_loss, guarda modelo ✅
3      0.60        0.75      NO mejora, contador=1

5      0.52        0.77      NO mejora, contador=3
...
10     0.40        0.78      NO mejora, contador=8
11     0.38        0.79      NO mejora, contador=9
12     0.36        0.80      contador=10 → ¡EARLY STOP! ⛔

Resultado: Usa modelo de época 2 (mejor val_loss)
```

```
Exp 1: Paciencia=10, paró en epoch 50 (mejor epoch)
Exp 2, Fase 1: Paciencia=10, paró en epoch 25

Exp 3: Paciencia=10, paró en epoch 59
Exp 4, Fase 2: Paciencia=10, paró en epoch 34
```

### ¿Qué es?
Guardar el modelo cada vez que mejora alguna métrica.

# CONCLUSIÓN

Las **métricas son historias** que cuenta el modelo:

1. **Loss** = "¿Cuán confiado estoy?" (pero equivocado es malo)
2. **Accuracy** = "¿Cuántas acerté?" (métrica más simple)
3. **Precision** = "¿Cuándo digo SÍ, tengo razón?" (evita falsas alarmas)
4. **Recall** = "¿Encuentro todos los casos reales?" (evita perdidas)
5. **F1** = "Balance entre precision y recall"
6. **Top-3** = "¿Lo saben mis top-3 candidatos?"

**Siempre lee MÚLTIPLES métricas, no solo una.**
